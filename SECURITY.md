# Security Policy

Lode is pre-alpha. While there are no released versions yet, please report any security
issue privately rather than opening a public issue.

- Use GitHub's **private vulnerability reporting** (Security → Report a vulnerability), or
- email the maintainer.

Please include reproduction steps and the affected version (`lode --version`). We will
acknowledge within a reasonable window and coordinate a fix and disclosure.

Note: logs are sensitive data. Lode redacts secrets/PII at ingest (RFC-0015); reports
about redaction gaps are in scope.
